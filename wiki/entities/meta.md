# Meta

## 摘要
Meta 在推荐系统与大模型融合领域扮演了关键的工业界引领者角色。2024 年，Meta AI 研究团队在 ICML 发表里程碑式工作，首次将大语言模型的生成式范式与 Scaling Law 成功迁移至推荐系统。通过将传统推荐任务重构为自回归序列生成问题，并设计专为高基数、非平稳流式数据优化的 **HSTU（Hierarchical Sequential Transduction Unit）** 架构，Meta 成功训练并部署了参数量达 **1.5 万亿** 的生成式推荐模型。该实践不仅打破了传统 DLRM 依赖手工特征拼接与双塔架构的瓶颈，更在十亿级用户平台上实证了推荐模型质量随训练算力呈严格幂律增长的规律，标志着推荐系统正式迈入“基础模型（Foundation Model）”原生时代。

## 核心要点
- **范式统一**：将召回、排序、重排统一为离散 Token 的自回归生成任务，消除多阶段流水线的信息损耗。
- **架构创新**：提出 HSTU 序列转换器，摒弃全局 Softmax 注意力，采用相对位置偏置与 GLU 门控机制，实现接近 $O(L)$ 的线性计算复杂度。
- **Scaling Law 实证**：首次验证推荐领域存在稳定的“算力-性能”幂律关系，模型规模跨越 3 个数量级（十亿至万亿）未出现性能饱和。
- **工业级部署**：1.5 万亿参数模型在真实业务 A/B 测试中核心转化指标提升 **12.4%**，推理吞吐量较 FlashAttention2 提升 5.3~15.2 倍。
- **流式训练策略**：针对推荐数据分布快速漂移特性，采用在线流式采样、动态学习率调度与大规模混合精度数据并行，保障万亿模型稳定收敛。

## 详细说明

### 1. 生成式推荐范式重构
传统工业推荐系统高度依赖稠密/稀疏特征拼接、多塔架构（如双塔召回+精排模型）以及复杂的特征工程流水线。Meta 提出将推荐任务彻底重构为**生成式建模框架下的序列转换问题**。在该范式下，用户历史交互序列（点击、购买、停留等）被直接离散化为共享词表中的 Token 序列，模型通过因果掩码（Causal Masking）进行自回归预测，逐 Token 生成下一个最可能交互的物品。这一设计不仅统一了召回、排序与重排任务，还通过高基数特征离散化技术消除了传统 Embedding 查找表的内存瓶颈，使模型能够直接处理超大规模稀疏空间，实现端到端的序列生成。

### 2. HSTU 架构与高效序列建模
为适配推荐场景序列长、分布漂移快、实时性要求高的特点，Meta 设计了 **HSTU（分层序列转换单元）** 作为生成式推荐的核心骨干网络。HSTU 采用多层堆叠结构，在架构层面进行了多项针对性优化：
- **线性化注意力机制**：移除计算密集的全局 Softmax 注意力，改用可学习的相对位置偏置替代绝对位置编码，结合低秩投影与分块计算策略，将序列建模复杂度从 $O(L^2)$ 优化至接近 $O(L)$。
- **门控前馈网络**：引入 GLU（Gated Linear Unit）增强非线性表征能力，在保持长程依赖捕获的同时大幅降低参数量与显存占用。
- **动态序列适配**：支持动态长度输入，能够高效处理长达 8192 的超长交互序列，满足工业场景毫秒级延迟与高吞吐推理需求。

### 3. 万亿参数部署与流式训练策略
Meta 在工业级数据流上完成了万亿参数模型的训练与部署验证。针对推荐数据非平稳、分布随时间快速变化的特性，团队采用了**在线流式采样与动态学习率调度**策略，结合混合精度训练与超大规模数据并行，保障了模型在持续流入数据上的稳定收敛。实验表明，模型质量随训练算力增加呈现严格的幂律增长曲线，跨越十亿至万亿参数规模未出现性能 Plateau。在真实业务环境中，基于 HSTU 构建的 1.5 万亿参数模型已稳定上线于多个十亿级用户平台，核心转化指标实现 **12.4%** 的显著提升，充分验证了“更大算力+更大模型”在推荐场景中的有效性。

### 4. 局限性与工程挑战
尽管生成式推荐范式展现出巨大潜力，但 Meta 的实践也暴露出若干现实挑战：
- **算力与工程门槛极高**：万亿参数训练与推理依赖超大规模分布式集群、定制化通信优化（如 Ring-AllReduce、张量并行）与专用硬件，中小团队难以低成本复现。
- **冷启动与长尾覆盖不足**：生成式范式高度依赖历史行为序列的上下文密度，对零交互新用户或极度稀疏的长尾物品预测能力较弱，仍需结合知识图谱、外部先验或传统特征工程进行增强。
- **可解释性与业务归因弱**：端到端自回归生成过程缺乏传统模型的特征权重可视化能力，在需要强业务归因、合规审计或人工策略干预的场景中应用受限。

## 开放问题
1. **算力平权路径**：如何通过知识蒸馏、稀疏激活（MoE）或低秩适配（LoRA）等技术，将万亿级生成式推荐模型的能力下沉至中小规模算力集群？
2. **冷启动融合机制**：如何在自回归生成框架中无缝注入知识图谱、多模态内容特征或人口统计学先验，以缓解零交互场景下的生成退化问题？
3. **可解释性增强**：能否在 HSTU 等生成式架构中引入注意力可视化、反事实推理或因果干预模块，以满足工业界对推荐决策透明度的合规要求？
4. **多模态序列扩展**：当前 HSTU 主要处理离散交互 Token，如何将其扩展至图文、视频等多模态连续 Token 的联合生成推荐场景？
