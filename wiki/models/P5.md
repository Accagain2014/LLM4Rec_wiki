# P5 — 基于提示的个性化预测

## 摘要

P5（Prompt-based Personalized Prediction）是一个统一的框架，它使用**单个预训练语言模型将所有推荐任务转化为文本生成**。通过设计**任务特定的提示模板**，P5 在一个模型内处理评分预测、序列推荐、评论生成和解释——无需为每个任务设计特定的架构。

## 要点

- **统一架构**：一个模型处理所有推荐任务
- **提示模板**：100+ 模板覆盖多种推荐系统任务
- **无任务特定参数**：完全依赖预训练语言模型知识
- **零样本/少样本能力**：可在无任务特定训练的情况下执行任务
- **开源**：基于 T5 模型家族

## 详情

### 核心思想

传统推荐系统为不同任务使用不同架构：
- 评分预测 → 矩阵分解
- 序列推荐 → RNN/Transformer
- 解释生成 → 单独的生成模型

P5 将所有内容统一：
```
All tasks → Text-to-text → Same model
```

### 提示模板示例

**评分预测：**
```
User {user_id} has rated:
- {item_1}: {rating_1} stars
- {item_2}: {rating_2} stars
How would User {user_id} rate {item_target}?
Answer: {rating} stars
```

**序列推荐：**
```
User {user_id} has interacted with in order:
{item_1} → {item_2} → {item_3}
What is the next item?
Answer: {item_next}
```

**带解释的推荐：**
```
User {user_id} likes: {item_1}, {item_2}
Recommend a new item.
Answer: {item_rec} because {explanation}
```

### 训练策略

1. **模板构建**：将每个训练样本转换为提示格式
2. **多任务训练**：在训练中混合所有任务模板
3. **微调**：在混合任务数据上更新 T5 权重
4. **推理**：从提示生成输出文本

### 性能表现

- 在评分预测上与任务特定模型具有竞争力
- 在未见过的任务上具有较强的零样本性能
- 少样本提示缩小了与微调模型之间的差距
- 在超大型物品目录上存在困难（ID 表示瓶颈）

### 局限性

- T5 模型规模限制了容量（与现代 LLM 相比）
- 基于 ID 的提示无法利用物品语义
- 提示长度随用户历史增长
- 未在对话式或交互式推荐上进行评估
