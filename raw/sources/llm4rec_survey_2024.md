# 综述：大语言模型用于推荐

**来源类型**：综述论文摘要
**年份**：2024
**主题**：LLM4Rec方法的综合综述

## 概述

本综述全面介绍了大语言模型如何应用于推荐系统。涵盖了从传统推荐系统到LLM增强和LLM原生方法的演进过程。

## 主要主题

### 1. 范式转变
- 传统推荐系统：基于ID、交互驱动、专用架构
- LLM4Rec：基于文本、知识增强、统一架构
- 动机：克服冷启动、提高可解释性、实现对话式推荐

### 2. LLM角色
- **增强器**：LLM为传统推荐系统提供特征/嵌入
- **预测器**：LLM直接输出推荐
- **解释器**：LLM为推荐生成解释
- **对话者**：LLM通过对话精炼推荐

### 3. 适配方法
- **提示工程**：设计提示，无需模型更新
- **提示微调**：学习软提示嵌入
- **LoRA/PEFT**：参数高效微调
- **全量微调**：更新所有模型权重

### 4. 关键挑战
- **效率**：LLM计算成本高昂
- **幻觉**：LLM可能生成不正确信息
- **评估**：传统指标无法捕捉LLM能力
- **隐私**：用户数据发送到LLM API

### 5. 未来方向
- 多模态LLM4Rec（文本+图像+音频）
- 智能体推荐（LLM作为自主推荐器）
- 联邦LLM4Rec（隐私保护）
- 实时LLM推荐

## 涵盖的重要论文

1. P5（Hou 等，2023）— 统一提示框架
2. InstructRec（Zhang 等，2023）— 指令微调
3. TALLRec（Bao 等，2023）— 高效适配
4. LLMRank（Tan 等，2024）— 列表级排序
5. ChatRec（2023）— 对话式推荐

## 要点

- LLM4Rec正在快速发展，但尚未在所有场景中达到生产就绪状态
- 混合方法（传统+LLM）目前最为实用
- 效率改进（蒸馏、PEFT）对部署至关重要
- 评估方法需要重大发展
