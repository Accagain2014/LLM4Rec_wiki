# 面向推荐系统的提示式微调

## 摘要

提示式微调通过**学习最优提示表示**和/或**更新模型参数的一个小**子集，将预训练语言模型适配到推荐任务中。与全量微调（更新所有参数）不同，这些方法是**参数高效**的——通常只更新不到 1% 的参数即可达到有竞争力的性能。这使得 LLM 的适配对不具备大规模计算资源的组织也变得切实可行。

## 要点

- **参数高效微调（PEFT）**方法：LoRA、Prefix-tuning、Prompt-tuning
- **P5** 开创了统一提示模板以覆盖所有推荐任务
- 只需更新少量参数即可获得良好的推荐性能
- **LoRA**（低秩适配）是 LLM4Rec 中最流行的方法
- 权衡：PEFT 更便宜，但在复杂任务上可能逊于全量微调

## 详情

### 为什么需要参数高效微调？

全量微调 LLM 成本高昂：
- Qwen-7B：70 亿参数 → 训练需要约 28GB 显存
- 训练数据需求：数百万条用户-项目交互
- **灾难性遗忘**预训练知识的风险

PEFT 方法通过仅更新一小部分参数来解决这个问题：

| 方法 | 更新内容 | 参数占比 | 典型用途 |
|--------|----------------|-------------|-------------|
| **LoRA** | 注意力层中的低秩矩阵 | 0.1-1% | 大多数 LLM4Rec 方法 |
| **Prefix-tuning** | 注意力中的连续前缀 | 0.01-0.1% | 序列推荐 |
| **Prompt-tuning** | 软提示嵌入 | <0.01% | 任务特定提示 |
| **Adapter 层** | 插入的小型神经模块 | 1-3% | 多任务推荐 |

### LoRA 在推荐中的应用

**工作原理：**
- 冻结预训练权重矩阵 W
- 学习低秩分解：ΔW = A × B，其中 A ∈ ℝ^(r×d), B ∈ ℝ^(d×r)
- 最终权重：W' = W + ΔW
- 典型秩 r = 8-64（相比 d = 4096+）

**用于推荐：**
```python
# 概念示例
lora_config = LoraConfig(
    task_type="CAUSAL_LM",
    r=16,                    # 低秩
    lora_alpha=32,           # 缩放因子
    lora_dropout=0.1,
    target_modules=["q_proj", "v_proj"]  # 注意力层
)
model = get_peft_model(base_model, lora_config)
```

**优势：**
- 单 GPU 即可训练
- 可为多个任务/域使用多个 LoRA 适配器
- 易于为不同推荐场景切换适配器

### 提示微调

学习**连续软提示**替代离散文本提示：

```
[软提示嵌入] + [用户历史 Token] + [任务指令] → LLM → 输出
```

- 软提示是任务特定的学习向量
- 比手动提示工程更高效
- 可为不同域学习不同的软提示

### 多任务微调

一种强大的方法：在**多个推荐任务**上微调一个模型：

```
任务混合：
  40% 评分预测
  30% 序列推荐
  20% 评论生成
  10% 解释生成
```

模型学习跨任务泛化的共享表示。

### 训练数据准备

**正例：**
- 用户-项目交互（点击、购买、评分）
- 格式化为提示："用户 X 喜欢项目 Y"

**负例：**
- 未观测到的交互（隐式负例）
- 困难负例：与喜欢的项目相似但未被选择的项目

**指令格式（InstructRec 风格）：**
```
指令：为该用户推荐一部电影。
输入：用户历史：[《盗梦空间》, 《星际穿越》, 《信条》]
输出：我推荐《降临》——它同样具有深思熟虑的科幻风格。
```

### 实践建议

| 场景 | 推荐方法 |
|----------|---------------------|
| 单域，数据充足 | LoRA 微调 |
| 多域 | 多任务 LoRA + 域适配器 |
| 数据较少 | 少样本提示（无需微调） |
| 算力受限 | 仅提示微调 |
| 生产部署 | 将微调后的 LLM 蒸馏为更小模型 |
