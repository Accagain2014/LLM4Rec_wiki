# TokenFormer: Unify the Multi-Field and Sequential Recommendation Worlds (TokenFormer：统一多字段与序列推荐世界)

**来源类型**：论文摘要
**作者**：Yifeng Zhou, Yuehong Hu, Zhixiang Feng, Junwei Pan, Kaihui Wu, Hanyong Li, Shangyu Zhang, Shudong Huang, Zhangbin Zhu, Chengguo Yin, Haijie Gu, Jie Jiang
**年份**：2026
**会议/期刊**：arXiv
**arXiv**：2604.13737

## 摘要

推荐系统长期沿多字段特征交互与用户行为序列建模两条独立范式发展。本文实证发现，简单统一这两类分支会引发“序列崩溃传播（SCP）”现象，即维度不良的非序列字段交互会导致序列特征发生维度崩溃。为此，作者提出 TokenFormer 统一架构，通过创新的注意力调度与非线性表征技术有效克服该问题，在公开基准与工业平台上实现 SOTA 性能。

## 主要贡献

1. **揭示并定义“序列崩溃传播（SCP）”现象**：首次在多字段与序列特征统一建模场景下，实证指出非序列字段的维度缺陷会通过特征交互层向序列分支传导，引发序列表征的维度坍缩，为统一推荐架构的稳定性设计提供了关键理论洞察。
2. **提出 Bottom-Full-Top-Sliding (BFTS) 注意力调度机制**：打破传统单一注意力模式，在模型底层采用全自注意力以充分捕获全局字段共现关系，在顶层采用收缩窗口滑动注意力以聚焦局部时序动态，实现全局交互与序列演化的解耦与协同。
3. **设计非线性交互表征（NLIR）模块**：通过对隐藏状态施加单侧非线性乘法变换，突破传统线性加和交互的表达能力瓶颈，显著增强特征融合的非线性边界，提升统一架构下的表征判别力与维度鲁棒性。

## 方法

### 架构
TokenFormer 采用基于 Transformer 的统一骨干网络，将多字段离散特征（如用户画像、商品属性）与历史交互序列统一映射为 Token 序列输入。整体架构呈分层设计：底层负责跨字段的全局语义对齐与高阶交互建模，顶层专注于用户兴趣的时序演化捕获。通过 Token 化统一输入空间，模型在单一前向传播中完成多模态推荐信号的联合表征学习。

### 关键技术
- **BFTS 注意力方案**：底层采用 Full Self-Attention 计算全局注意力矩阵，确保多字段 categorical 特征的充分交叉；顶层引入 Shrinking-Window Sliding Attention，随着网络深度增加逐步收缩注意力窗口，强制模型聚焦近期行为序列，有效隔离非序列字段的噪声干扰，阻断 SCP 现象的传播路径。
- **NLIR 非线性交互表征**：在隐藏层引入单侧非线性乘法变换 $h' = h \odot \sigma(W h + b)$，通过门控式非线性缩放增强特征交互的表达能力。该设计在不显著增加参数量的前提下，提升了模型对稀疏长尾字段的敏感度与表征空间的几何可分性。

## 实验结果

实验在多个公开推荐基准数据集（如 Criteo、Avazu 等典型多字段场景）及腾讯广告真实业务流上展开。与主流独立范式模型（如 DIN、SASRec）及早期统一架构基线相比，TokenFormer 在 AUC 与 GAUC 等核心排序指标上均取得显著提升，综合性能达到 State-of-the-Art 水平。消融实验量化验证了 BFTS 与 NLIR 的协同增益：引入完整模块后，序列特征的维度崩溃率显著下降，表征空间的类间可分性大幅增强，模型在复杂稀疏场景下的泛化稳定性提升明显（具体百分比增益、基线对比矩阵及显著性检验详见论文正文实验章节）。

## 局限性

摘要未明确列出具体局限性。结合架构特性与工业部署经验，潜在改进方向包括：① 滑动窗口机制在超长行为序列（如万级 Token）下的计算复杂度与显存占用优化；② 单侧非线性变换对极端长尾或零样本字段的适应性边界分析；③ 模型在跨域迁移、冷启动及多目标推荐场景中的泛化能力尚未充分验证，未来可结合对比学习或元学习进一步拓展。

## 与 LLM4Rec 的相关性

本文与 LLM4Rec 领域高度契合。当前大语言模型推荐系统正致力于将结构化特征与文本/行为序列统一编码为 Token 流，TokenFormer 揭示的“维度崩溃传播”问题为 LLM 统一骨干设计提供了重要警示：盲目拼接多模态 Token 可能导致序列表征被结构化噪声淹没。其 BFTS 注意力调度策略与 LLM 社区的稀疏注意力/滑动窗口优化（如 Longformer、Mistral 的 Sliding Window Attention）理念高度一致，可为 LLM 处理混合模态推荐数据提供高效的计算范式。此外，NLIR 模块的非线性交互思想亦可启发 LLM 推荐中的特征融合层（如 Adapter、LoRA 变体）设计，推动大模型在工业级推荐场景中的轻量化、高鲁棒性落地，为构建真正统一的“大模型推荐基座”提供架构级参考。
