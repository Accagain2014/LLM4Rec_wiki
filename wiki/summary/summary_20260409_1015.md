## 回答
# 行为数据在生成式召回中的应用

## 直接回答

在生成式召回（Generative Retrieval）中，**用户行为数据被序列化为文本格式的 token 序列**，作为 Transformer 模型的输入上下文，模型通过**自回归生成目标物品的 Semantic ID**来完成召回。这与传统"嵌入 +ANN 检索"范式根本不同——生成式召回将召回任务转化为**序列到序列的生成任务**[来源：[concepts/generative_retrieval.md](wiki/concepts/generative_retrieval.md)]。

---

## 一、行为数据的编码与序列化

### 1. 行为序列格式化

用户行为历史被转换为**有序的 token 序列**，典型格式如下：

```
User Context → [Item_ID₁, Item_ID₂, ..., Item_IDₙ] → Transformer → Semantic ID
```

- **序列推荐场景**：预测用户会话中的下一个物品 Semantic ID[来源：[concepts/sequential_recommendation.md](wiki/concepts/sequential_recommendation.md)]
- **时间信息编码**：可加入时间戳、会话边界等 temporal 信号[来源：[concepts/sequential_recommendation.md](wiki/concepts/sequential_recommendation.md)]
- **多模态行为**：点击、观看时长、购买等不同交互类型可编码为不同 token 类型[来源：[sources/2604_paper_26040497_Tencent_Advertising_Algorithm_Challenge_2025_All-Modality_G.md](wiki/sources/2604_paper_26040497_Tencent_Advertising_Algorithm_Challenge_2025_All-Modality_G.md)]

### 2. 与传统方法的区别

| 方面 | 传统召回 | 生成式召回 |
|------|---------|-----------|
| **行为表示** | 用户嵌入向量 | Token 序列 |
| **物品表示** | 物品嵌入向量 | Semantic ID（离散码字元组） |
| **匹配方式** | 向量相似度（ANN） | 自回归生成 |
| **泛化能力** | 依赖共现统计 | 语义泛化到未见物品 |

[来源：[`concepts/generative_retrieval.md`](wiki/concepts/generative_retrieval.md), `models/DSI.md`]

## 二、Semantic ID：行为与物品的桥梁

### 1. Semantic ID 的构建

Semantic ID 是生成式召回的核心创新，将物品映射为**有意义的离散 token 序列**：

```
Item → Encoder → Continuous Embedding → Quantizer → [c₁, c₂, ..., cₖ]
```

- **层次化结构**：前缀码控制粗粒度（类目），后缀码控制细粒度（具体物品）[来源：[concepts/semantic_id.md](wiki/concepts/semantic_id.md)]
- **语义相似性**：相似物品共享前缀码字，支持语义泛化 [来源：[models/DSI.md](wiki/models/DSI.md)]
- **构建方法**：RQ-VAE、FORGE（力引导探索）、对比自编码器等 [来源：[[`concepts/semantic_id.md`](wiki/concepts/semantic_id.md), [`sources/2509_paper_25092090_FORGE_Forming_Semantic_Identifiers_for_Generative_Retrieval.md`](wiki/sources/2509_paper_25092090_FORGE_Forming_Semantic_Identifiers_for_Generative_Retrieval.md)](wiki/sources/2509_paper_25092090_FORGE_Forming_Semantic_Identifiers_for_Generative_Retrieval.md)]

### 2. 行为数据与 Semantic ID 的关联

训练目标是学习从**用户行为序列**到**目标物品 Semantic ID**的映射：

```
训练样本：
输入：[用户历史物品 Semantic ID 序列]
输出：[下一个物品的 Semantic ID]
损失函数：交叉熵损失（在目标 Semantic ID 上）
```

[来源：[`models/DSI.md`, [`sources/paper_4ddaf2_Recommender_Systems_with_Generative_Retrieval.md`](wiki/sources/paper_4ddaf2_Recommender_Systems_with_Generative_Retrieval.md)](wiki/sources/paper_4ddaf2_Recommender_Systems_with_Generative_Retrieval.md)]

## 三、模型架构与训练目标

### 1. 主流架构

| 模型 | 架构类型 | 行为数据处理方式 |
|------|---------|-----------------|
| **DSI/TIGER** | Encoder-Decoder | 用户历史编码，自回归解码物品 ID[来源：[models/DSI.md](wiki/models/DSI.md)] |
| **PLUM** | Decoder-only + CPT | 持续预训练适配领域行为模式 [来源：[models/PLUM.md](wiki/models/PLUM.md)] |
| **OneRec** | Encoder-Decoder + MoE | 会话级生成，统一检索 + 排序 [来源：[models/OneRec.md](wiki/models/OneRec.md)] |
| **HiGR** | 层次化规划 | 列表级意图规划→物品级解码 [来源：[models/HiGR.md](wiki/models/HiGR.md)] |

### 2. 训练目标

生成式召回的核心训练目标：

$$\mathcal{L} = -\sum_{t=1}^{T} \log P(\text{Semantic ID}_t | \text{User Behavior}_{<t}, \text{Semantic ID}_{<t})$$

- **自回归生成**：逐个 token 生成目标物品的 Semantic ID[来源：[concepts/generative_retrieval.md](wiki/concepts/generative_retrieval.md)]
- **会话级生成**：OneRec 提出一次性生成整个推荐会话，而非逐点预测 [来源：[[`models/OneRec.md`](wiki/models/OneRec.md), [`concepts/session_wise_generation.md`](wiki/concepts/session_wise_generation.md)](wiki/concepts/session_wise_generation.md)]
- **偏好对齐**：结合 DPO 等强化学习方法对齐真实用户反馈 [来源：[[`models/OneRec.md`](wiki/models/OneRec.md), [`methods/iterative_preference_alignment.md`](wiki/methods/iterative_preference_alignment.md)](wiki/methods/iterative_preference_alignment.md)]

## 四、工业实践案例

### 1. 快手 OneRec 系列

- **行为序列长度**：支持长用户行为历史编码 [来源：[models/OneRec.md](wiki/models/OneRec.md)]
- **部署效果**：快手主场景观看时长 +1.6%[来源：[models/OneRec.md](wiki/models/OneRec.md)]
- **OneRec-V2**：惰性纯解码器架构，计算量降低 94%，扩展到 8B 参数 [来源：[[`models/OneRec-V2.md`](wiki/models/OneRec-V2.md), [`sources/2508_paper_25082090_OneRec-V2_Technical_Report.md`](wiki/sources/2508_paper_25082090_OneRec-V2_Technical_Report.md)](wiki/sources/2508_paper_25082090_OneRec-V2_Technical_Report.md)]

### 2. 腾讯 HiGR

- **层次化规划**：列表级意图→物品级解码，减少搜索空间 [来源：[models/HiGR.md](wiki/models/HiGR.md)]
- **部署效果**：腾讯商业平台，观看时长 +1.22%，播放量 +1.73%[来源：[[`models/HiGR.md`](wiki/models/HiGR.md), [`sources/paper_8edbf8_HiGR_Efficient_Generative_Slate_Recommendation_via_Hierarch.md`](wiki/sources/paper_8edbf8_HiGR_Efficient_Generative_Slate_Recommendation_via_Hierarch.md)](wiki/sources/paper_8edbf8_HiGR_Efficient_Generative_Slate_Recommendation_via_Hierarch.md)]

### 3. Google/YouTube PLUM

- **持续预训练**：在领域行为数据上 CPT 适配推荐模式 [来源：[models/PLUM.md](wiki/models/PLUM.md)]
- **部署规模**：YouTube 数十亿用户 [来源：[[`models/PLUM.md`](wiki/models/PLUM.md), [`sources/paper_81ec38_PLUM_Adapting_Pre-trained_Language_Models_for_Industrial-sc.md`](wiki/sources/paper_81ec38_PLUM_Adapting_Pre-trained_Language_Models_for_Industrial-sc.md)](wiki/sources/paper_81ec38_PLUM_Adapting_Pre-trained_Language_Models_for_Industrial-sc.md)]

### 4. 腾讯广告挑战赛数据集

- **TencentGR-1M/10M**：100 万/1000 万用户行为序列，包含曝光、点击、转化信号 [来源：[[`sources/2604_paper_26040497_Tencent_Advertising_Algorithm_Challenge_2025_All-Modality_G.md`](wiki/sources/2604_paper_26040497_Tencent_Advertising_Algorithm_Challenge_2025_All-Modality_G.md), [`entities/tencentgr_dataset.md`](wiki/entities/tencentgr_dataset.md)](wiki/entities/tencentgr_dataset.md)]
- **全模态行为**：协同 ID+ 多模态表示（文本、图像等）[来源：[concepts/all_modality_gr.md](wiki/concepts/all_modality_gr.md)]

## 六、知识库中的空白与不确定性

1. **具体行为编码细节**：知识库未详细说明不同行为类型（点击 vs 观看时长 vs 购买）在 token 层面的具体编码差异 [来源：[`concepts/sequential_recommendation.md`](wiki/concepts/sequential_recommendation.md) 标注为草稿]

2. **序列长度限制**：不同模型支持的最大行为序列长度未统一说明，LONGER 提到支持 10k 序列但生成式召回的具体限制需进一步确认 [来源：[[`models/LONGER.md`](wiki/models/LONGER.md), [`methods/long_context_efficiency.md`](wiki/methods/long_context_efficiency.md)](wiki/methods/long_context_efficiency.md)]

3. **负样本处理**：生成式召回中如何构造和利用负样本行为数据的细节未充分披露 [来源：[`methods/reward_modeling_rec.md`](wiki/methods/reward_modeling_rec.md) 标注为草稿]

4. **实时行为更新**：用户实时行为如何增量更新到生成模型中的策略未详细说明 [来源：[`concepts/generative_retrieval.md`](wiki/concepts/generative_retrieval.md) 开放问题部分]

## 总结

行为数据在生成式召回中的应用核心是：**将用户行为序列化为 token 输入，通过自回归生成目标物品的 Semantic ID**。这一范式由 Google 在 NeurIPS 2023 首次引入推荐系统，随后在快手（OneRec）、腾讯（HiGR）、YouTube（PLUM）等工业场景得到验证，实现了从"检索然后排序"到"生成即推荐"的范式转变 [来源：[`concepts/generative_retrieval.md`](wiki/concepts/generative_retrieval.md), `models/DSI.md`]。
