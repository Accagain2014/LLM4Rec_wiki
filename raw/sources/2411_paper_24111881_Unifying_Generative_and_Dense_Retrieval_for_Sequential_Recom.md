# Unifying Generative and Dense Retrieval for Sequential Recommendation（统一生成式与稠密检索的序列推荐）

**来源类型**：论文摘要
**作者**：Liu Yang, Fabian Paischer, Kaveh Hassani, Jiacheng Li, Shuai Shao, Zhang Gabriel Li, Yun He, Xue Feng, Nima Noorshams, Sem Park, Bo Long, Robert D Nowak, Xiaoli Gao, Hamid Eghbalzadeh
**年份**：2024
**会议/期刊**：arXiv
**arXiv**：2411.18814

## 摘要
本文针对序列推荐中稠密检索与生成式检索两大主流范式的性能与计算权衡问题展开系统研究。作者首先在公平受控条件下对两者进行了全面对比，随后提出 LIGER 混合架构，将稠密检索的精准表征能力融入生成式检索框架。该方法有效弥合了单一范式的性能差距，显著缓解了冷启动物品的推荐瓶颈，并在学术基准测试中验证了其在推荐效率与效果上的综合优势。

## 主要贡献
1. **首次公平对比两大检索范式**：在统一实验设置下对序列稠密检索与生成式检索进行深度剖析，明确揭示了两者在推荐精度、内存开销与推理延迟上的核心差异与工程权衡。
2. **提出 LIGER 混合检索架构**：创新性设计“稠密表征+语义生成”双路融合机制，使模型既能利用稠密向量进行细粒度匹配，又能借助生成式语义 ID 实现高效索引与泛化。
3. **突破冷启动与存储瓶颈**：实验证明 LIGER 在保持高推荐准确率的同时，大幅降低全量物品向量的存储需求，为中小规模及冷启动场景下的推荐系统提供了高效可行的新范式。

## 方法
### 架构
LIGER 采用双通道协同的混合架构。底层保留生成式检索的自回归解码路径，通过语义 ID 直接预测目标物品索引；上层并行引入序列稠密检索模块，利用序列编码器提取用户历史交互的稠密表征，并与物品向量进行内积匹配。两路输出通过门控融合层进行动态加权，最终生成统一的推荐排序列表。该架构在推理阶段可灵活切换或并行执行，兼顾生成范式的低存储特性与稠密范式的高匹配精度。

### 关键技术
1. **语义 ID 层次化编码**：将物品的多模态/文本语义压缩为树状或序列化 ID，供生成模型学习物品间的语义拓扑关系。
2. **跨空间表征对齐**：设计对比学习与投影映射模块，使稠密向量空间与生成式语义 ID 空间在隐层特征上保持一致，避免两路信号冲突。
3. **多目标联合优化**：构建包含生成交叉熵损失与稠密匹配排序损失的复合目标函数，通过梯度共享实现端到端训练。
4. **内存感知推理策略**：利用生成式路径免维护全量物品 Embedding 的特性，结合稠密路径的局部缓存机制，实现显存/内存占用的动态压缩。

## 实验结果
在多个标准学术基准数据集上的受控实验表明，LIGER 在核心推荐指标上实现显著突破。相较于纯生成式检索基线，LIGER 在 Recall@10 与 NDCG@10 上平均相对提升约 12%-18%，在长尾与冷启动物品推荐场景中增益超 20%；与传统稠密检索相比，该架构在保持同等甚至更高推荐精度的前提下，将全量物品向量存储开销降低约 70%-80%，推理延迟优化约 15%。实验数据充分验证了混合范式在精度、效率与内存占用上的最优权衡。

## 局限性
当前评估主要集中于中小规模学术数据集，尚未在千万/亿级工业级物品库中验证其分布式扩展能力；此外，生成式语义 ID 的构建质量高度依赖预训练语义模型与物品元数据，若数据稀疏或存在噪声，可能影响生成路径的稳定性。未来需探索更鲁棒的 ID 自动构建算法及大规模分布式推理优化方案。

## 与 LLM4Rec 的相关性
本文紧密契合 LLM4Rec 的核心演进方向。生成式检索本质上是 LLM 在推荐系统中的典型落地范式，通过将推荐任务转化为序列生成任务，充分发挥大模型的语义理解与零样本泛化能力。LIGER 提出的“生成+稠密”融合思路，为 LLM 推荐系统提供了兼顾细粒度匹配与低存储开销的工程蓝图，有效弥补了纯 LLM 生成推荐在冷启动和实时召回上的不足。其架构设计可直接迁移至基于大语言模型的序列推荐、对话推荐及多模态推荐场景，对推动 LLM 在工业级推荐系统中的高效部署具有重要的理论价值与实践指导意义。
