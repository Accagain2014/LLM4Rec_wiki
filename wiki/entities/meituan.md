# MBGR — 美团多业务生成式推荐框架

## 摘要
MBGR（Multi-Business Prediction for Generative Recommendation）是美团于2026年提出的首个面向工业级多业务场景的生成式推荐（Generative Recommendation, GR）框架。该框架旨在解决大语言模型（LLM）的 Next Token Prediction（NTP）范式在跨业务推荐中面临的“跷跷板效应”与“表征混淆”两大核心瓶颈。通过引入业务感知语义ID（BID）、多业务预测结构（MBP）与标签动态路由（LDR）三大核心模块，MBGR实现了跨业务行为模式的有效解耦与细粒度定制化生成。该模型已在美团外卖平台完成全量部署，显著提升了点击率（CTR）、转化率（CVR）及整体GMV，为“超级App”架构下的统一推荐大模型提供了可落地的工业范式。

## 核心要点
- **业务感知语义ID（BID）**：摒弃全局统一词表，采用领域感知分词策略构建解耦的语义子空间，从根本上消除跨业务表征混淆。
- **多业务预测结构（MBP）**：在自回归解码中引入业务条件门控与动态路由机制，替代单一全局损失函数，缓解多任务优化中的梯度冲突与“跷跷板现象”。
- **标签动态路由（LDR）**：将稀疏的跨业务交互标签转化为稠密监督信号，结合辅助对比学习，显著提升长尾业务覆盖与多任务协同训练稳定性。
- **工业级验证**：在美团外卖真实场景完成离线评估与在线A/B测试，核心业务指标全面正向提升，推理延迟满足实时推荐要求，已实现生产环境全量上线。

## 详细说明

### 1. 背景与动机：多业务生成式推荐的工业挑战
随着推荐系统向“超级App”演进，单一模型需同时服务外卖、到店、酒旅等多个业务线。传统生成式推荐（GR）通常依赖统一的语义ID空间与全局 Next Token Prediction 目标。然而，在跨业务场景下，不同业务的数据分布、用户意图与物品属性差异巨大，直接套用统一范式会导致两大问题：
- **表征混淆（Representation Confusion）**：全局共享的 Tokenizer 无法区分业务上下文，导致不同业务的物品在隐空间中发生语义重叠，模型难以精准捕捉业务特异性。
- **跷跷板效应（Seesaw Effect）**：单一损失函数优化多业务目标时，强势业务（如高频外卖）的梯度会主导参数更新，挤压弱势或长尾业务（如低频酒旅）的优化空间，造成指标此消彼长。
MBGR 正是为破解上述工业痛点而生，旨在保留生成式推荐扩展性（Scaling Law）的同时，实现多业务场景下的精准路由与协同建模。

### 2. 核心架构与关键技术
MBGR 采用基于自回归生成的端到端架构，整体流程包含三个核心模块：
- **业务感知语义ID（BID）模块**：突破传统统一语义ID空间的局限，为不同业务线构建独立的语义子空间。通过共享底层基础词表与注入业务专属标识符（Business-specific Identifiers），实现语义隔离与跨域知识迁移。该设计确保模型在生成每一步时，能准确识别当前业务上下文，避免跨域语义干扰。
- **多业务预测（MBP）结构**：针对标准 NTP 难以捕捉复杂跨业务行为模式的缺陷，MBGR 在解码器层引入业务条件门控机制。在自回归生成过程中，模型根据目标业务动态激活对应的预测分支，实现细粒度的业务定制化生成。该机制有效解耦了各业务线的优化目标，避免了梯度冲突，使多业务协同训练更加稳定。
- **标签动态路由（LDR）模块**：工业场景中用户跨业务交互数据极度稀疏。LDR 模块通过动态路由算法，将离散的稀疏点击/转化标签映射为连续的稠密监督信号。结合辅助对比学习损失（Auxiliary Contrastive Loss），该策略不仅缓解了数据稀疏性对生成质量的负面影响，还增强了模型对冷启动物品与长尾业务的泛化能力。

### 3. 工业实践与实验效果
MBGR 在美团外卖平台进行了大规模离线评估与在线 A/B 测试。
- **离线实验**：在 Recall@50 与 NDCG@50 等核心排序指标上，MBGR 显著优于传统双塔模型及现有生成式推荐基线（如 P5、TIGER 等变体）。在多业务交叉场景下，指标提升幅度达到统计显著水平，验证了架构解耦的有效性。
- **在线 A/B 测试**：模型成功部署至生产环境，核心业务指标（CTR、CVR）均取得明确正向提升，整体 GMV 实现稳定增长。系统推理延迟经过工程优化，完全满足工业级实时推荐（<50ms）的 SLA 要求，充分证明了该框架在超大规模多业务推荐系统中的工程可行性与商业价值。

### 4. 局限性与未来优化方向
尽管 MBGR 在多业务协同生成方面取得突破，但仍存在以下局限：
- **极端长尾与零样本泛化**：当前架构主要依赖历史交互数据进行路由与监督，对于全新业务冷启动或极端长尾场景的零样本（Zero-shot）泛化能力尚未深入验证。
- **计算与存储开销**：BID 与 MBP 模块的引入增加了模型参数量与训练复杂度。在资源受限的边缘部署或超低延迟场景中，仍需结合模型压缩、知识蒸馏、量化（如 INT8/FP8）及推理加速技术进行进一步优化。
- **多模态扩展**：当前框架主要聚焦于结构化/文本化行为序列，未来可探索将全模态内容（图像、视频、地理位置）无缝映射至业务感知语义空间，以支持更丰富的生成式推荐场景。
