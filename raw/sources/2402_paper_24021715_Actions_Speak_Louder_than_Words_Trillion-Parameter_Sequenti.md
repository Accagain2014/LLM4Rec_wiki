# Actions Speak Louder than Words: Trillion-Parameter Sequential Transducers for Generative Recommendations (行动胜于言语：用于生成式推荐的万亿参数序列转换器)

**来源类型**：论文摘要
**作者**：Jiaqi Zhai, Lucy Liao, Xing Liu, Yueming Wang, Rui Li, Xuan Cao, Leon Gao, Zhaojie Gong, Fangda Gu, Michael He, Yinghai Lu, Yu Shi
**年份**：2024
**会议/期刊**：ICML'24
**arXiv**：arXiv:2402.17152

## 摘要
本文针对工业界深度推荐模型（DLRM）难以随算力线性扩展的瓶颈，提出将推荐任务重构为生成式建模框架下的序列转换问题（Generative Recommenders）。为此，作者设计了专为高基数、非平稳流式数据优化的新型架构 HSTU。该模型在万亿参数规模下展现出卓越的性能与扩展性，已成功部署于十亿级用户平台，并首次实证了推荐模型质量随训练算力呈幂律增长的规律，为推荐系统基础模型的演进奠定基石。

## 主要贡献
1. **提出生成式推荐新范式**：突破传统推荐系统依赖稠密/稀疏特征拼接与双塔/多塔架构的局限，将用户历史交互序列直接视为离散 Token 序列，在自回归生成框架下统一建模召回、排序与重排任务。
2. **设计高效序列转换器 HSTU**：针对推荐场景序列长、分布漂移快的特点，提出分层序列转换单元（HSTU）。该架构摒弃全局 Softmax 注意力，采用相对位置偏置与门控机制，在保持长程依赖捕获能力的同时大幅降低计算开销。
3. **验证推荐领域的 Scaling Law**：首次在推荐系统中实证模型性能与训练算力之间存在稳定的幂律关系，跨越三个数量级（十亿至万亿参数）未出现性能饱和，证明“更大算力+更大模型”可稳定提升推荐效果。

## 方法
### 架构
HSTU（Hierarchical Sequential Transduction Unit）是生成式推荐的核心骨干网络。其采用因果掩码的多层堆叠结构，将用户行为序列映射为高维隐状态后，通过层级转换单元进行上下文聚合。与传统 Transformer 不同，HSTU 移除了计算密集的全局注意力层，改用基于相对位置编码的局部-全局混合注意力机制，并引入 GLU 门控前馈网络增强非线性表征。整体架构支持动态序列长度输入，能够以自回归方式逐 Token 预测下一个交互物品，实现端到端的序列生成。

### 关键技术
- **相对位置偏置与线性化注意力**：采用可学习的相对位置偏置替代绝对位置编码，结合低秩投影与分块计算策略，将序列建模复杂度从 $O(L^2)$ 优化至接近 $O(L)$，显著提升长序列训练与推理效率。
- **高基数特征离散化与统一词表**：将海量用户 ID、物品 ID 及异构上下文特征统一离散化为共享词表中的 Token，消除传统 Embedding 查找表的内存瓶颈，使模型能够直接处理超大规模稀疏空间。
- **流式非平稳数据训练策略**：针对推荐数据分布随时间快速变化的特性，采用在线流式采样与动态学习率调度，结合混合精度训练与大规模数据并行，保障万亿参数模型在工业级数据流上的稳定收敛与泛化。

## 实验结果
- **离线性能对比**：在合成数据集与多个公开推荐基准上，HSTU 相比传统 DLRM、SASRec 及基于 FlashAttention2 的标准 Transformer，在 NDCG 指标上最高提升 **65.8%**，显著验证了生成式范式在序列建模上的优势。
- **推理效率优势**：在处理长度为 **8192** 的超长交互序列时，HSTU 的推理吞吐量达到基于 FlashAttention2 的 Transformer 的 **5.3 倍至 15.2 倍**，显存占用大幅降低，满足工业场景毫秒级延迟要求。
- **在线业务增益**：基于 HSTU 构建的 **1.5 万亿参数** 生成式推荐模型在真实业务 A/B 测试中，核心转化指标提升 **12.4%**，并已在多个十亿级用户平台表面稳定上线。
- **扩展性验证**：模型质量随训练算力增加呈现严格的幂律增长曲线，跨越 **3 个数量级**（从十亿级至万亿级参数），未出现性能 plateau，证实了 Scaling Law 在推荐领域的普适性。

## 局限性
- **算力与工程门槛极高**：万亿参数训练与推理依赖超大规模分布式集群、定制化通信优化与专用硬件，中小团队难以直接复现或低成本部署。
- **冷启动与长尾覆盖挑战**：生成式范式高度依赖历史行为序列的上下文密度，对于零交互新用户或极度稀疏的长尾物品，仍需结合传统特征工程、知识图谱或外部先验进行增强。
- **可解释性与业务归因弱**：端到端自回归生成过程缺乏传统推荐模型的特征权重可视化能力，在需要强业务归因、合规审计或策略干预的场景中应用受限。

## 与 LLM4Rec 的相关性
本文是 LLM4Rec 领域的里程碑式工作，首次将大语言模型的 Scaling Law 与生成式架构成功迁移至推荐系统。通过证明推荐模型同样遵循“算力-性能”幂律关系，彻底打破了传统推荐系统依赖手工特征工程的范式，推动推荐架构向“统一基础模型（Foundation Model）”演进。HSTU 在相对位置编码、线性注意力优化与离散 Token 化方面的设计，为后续 LLM 与推荐系统的深度融合提供了高效、可扩展的底层架构参考，标志着推荐系统正式迈入大模型原生时代。
