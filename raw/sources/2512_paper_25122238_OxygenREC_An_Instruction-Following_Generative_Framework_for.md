# OxygenREC: An Instruction-Following Generative Framework for E-commerce Recommendation (OxygenREC：一种遵循指令的电商推荐生成式框架)

**来源类型**：论文摘要
**作者**：Xuegang Hao, Ming Zhang, Alex Li, 等（共33位作者）
**年份**：2025
**会议/期刊**：arXiv
**arXiv**：2512.22386

## 摘要

针对传统推荐系统多阶段优化目标不一致、现有生成式推荐缺乏复杂意图推理能力，以及大语言模型在工业场景中延迟高、多场景扩展性差等痛点，本文提出OxygenREC框架。该框架创新性地引入“快慢思考”双轨架构，结合近线大模型深度推理与高效编码器-解码器实时生成，通过指令引导检索、语义对齐损失与多场景策略优化技术，实现了低延迟、强推理与“一次训练、处处部署”的工业级电商推荐系统。

## 主要贡献

1. **提出“快慢思考”双轨生成架构**：慢思考利用近线LLM流水线合成上下文推理指令，挖掘需演绎推理的复杂用户意图；快思考采用轻量高效的Encoder-Decoder骨干网络进行实时推荐生成，有效平衡了深度推理能力与工业级严格延迟要求。
2. **设计指令引导的语义对齐机制**：引入指令引导检索（IGR）动态过滤与意图高度相关的历史行为，并结合Query-to-Item（Q2I）损失函数，强制模型学习推理指令与目标物品的语义映射，显著提升生成结果的可解释性与准确性。
3. **实现多场景可扩展的统一训练范式**：将差异化场景信息转化为可控指令，通过统一奖励映射与软自适应组裁剪策略优化（SA-GCPO）对齐多样化业务目标，彻底解决传统方案需独立训练部署的瓶颈，实现“一次训练、处处部署”。

## 方法

### 架构
模型采用异步协同的“快慢思考（Fast-Slow Thinking）”架构。慢思考模块部署于近线环境，利用大语言模型的领域知识与逻辑演绎能力，结合用户实时上下文生成结构化“上下文推理指令”；快思考模块部署于在线服务，采用高吞吐、低延迟的Encoder-Decoder生成式骨干网络，直接接收慢思考输出的指令进行物品序列生成，满足电商场景毫秒级响应与高并发需求。

### 关键技术
- **指令引导检索（IGR）与Q2I一致性损失**：IGR机制根据推理指令的语义向量，在用户历史行为库中检索Top-K相关交互，构建精准上下文窗口；Q2I损失函数在训练阶段约束指令表征与目标物品表征的分布对齐，防止生成偏离用户真实意图。
- **多场景策略优化（SA-GCPO）**：将不同业务线（如首页推荐、搜索推荐、活动会场）的差异化目标抽象为统一奖励信号。SA-GCPO算法通过软裁剪机制动态调整策略梯度，避免多目标优化中的梯度冲突，使单一模型能够自适应不同场景的业务KPI。

## 实验结果

论文在真实电商数据集与线上A/B测试中验证了OxygenREC的工业有效性。相较于传统多阶段推荐基线与独立部署的生成式方案，OxygenREC在保持极低在线推理延迟（满足工业实时性阈值）的同时，显著提升了复杂意图下的推荐准确率。在多场景扩展性对比中，该框架将原本需独立训练与部署的多个场景模型统一为单一架构，大幅降低了算力开销与运维成本。具体量化指标（如CTR/CVR提升百分比、延迟降低毫秒数、多场景统一部署的GPU节省比例等）在论文完整实验章节中详细列出，整体性能在延迟、准确率与资源利用率三个维度上均优于现有SOTA方法。

## 局限性

1. **近线LLM推理的算力依赖**：慢思考模块依赖大语言模型生成推理指令，尽管采用近线异步处理，但在流量洪峰或数据分布剧烈漂移时，仍可能面临指令生成队列积压与算力瓶颈。
2. **指令噪声的级联放大风险**：快思考模块的生成质量高度依赖慢思考输出的指令准确性。若上下文推理指令存在语义偏差或幻觉，可能通过Q2I损失与生成过程被放大，影响最终推荐结果。
3. **极端多目标冲突下的策略平衡**：SA-GCPO在常规多场景下表现优异，但在强冲突业务目标（如短期转化率与长期用户留存）下的动态权重自适应能力仍需进一步理论验证与调优。

## 与 LLM4Rec 的相关性

本文是LLM4Rec（大语言模型赋能推荐系统）领域迈向工业落地的关键实践。它突破了现有LLM推荐方案“重推理、轻延迟”与“单场景定制”的局限，创造性地将LLM的深度演绎推理能力与传统生成式推荐的实时性相结合。通过“快慢思考”架构与指令对齐技术，为LLM在电商推荐中的部署提供了可解释、低延迟、高可扩展的新范式。该工作不仅推动了生成式推荐从“归纳模式匹配”向“演绎意图推理”演进，也为大模型在推荐系统中的分层部署、指令微调与多目标对齐提供了可复用的工程蓝图，对学术界与工业界均具有重要参考价值。
