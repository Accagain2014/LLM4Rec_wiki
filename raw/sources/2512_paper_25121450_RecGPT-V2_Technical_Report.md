# RecGPT-V2 Technical Report (RecGPT-V2 技术报告)

**来源类型**：论文摘要
**作者**：Chao Yi, Dian Chen, Gaoyang Guo, Jiakai Tang, Jian Wu 等（共35位作者）
**年份**：2025
**会议/期刊**：arXiv
**arXiv**：2512.14503

## 摘要
本文系统介绍了 RecGPT-V2，旨在突破初代 LLM 推荐模型在推理效率、解释多样性、监督泛化能力及评估标准上的四大瓶颈。通过构建分层多智能体协同架构、混合表示推理、动态元提示框架、约束强化学习及智能体裁判机制，该模型实现了从隐式行为匹配向显式意图推理的范式升级。在淘宝大规模在线 A/B 测试中，RecGPT-V2 显著降低了计算开销，并在点击率、页面访问量及新客探索等核心商业指标上取得全面突破，验证了大语言模型驱动意图推理在工业级推荐场景中的技术可行性与规模化商业价值。

## 主要贡献
1. **分层多智能体与混合表示推理架构**：重构意图推理链路，通过多智能体协同分工消除认知冗余，结合用户行为上下文压缩技术，在保障意图覆盖广度的同时实现算力的大幅优化。
2. **动态元提示与约束强化学习框架**：提出上下文自适应的元提示生成机制，打破固定模板的生成僵化问题；引入约束强化学习有效缓解多目标优化中的奖励冲突，显著提升标签预测精度与解释可接受度。
3. **多步推理评估体系与工业级落地验证**：构建 Agent-as-a-Judge 框架将传统结果导向评估升级为多步逻辑推理评估，更精准对齐人类偏好；在真实电商流量中完成全链路部署，证明其具备高并发、低延迟的工业可用性。

## 方法
### 架构
RecGPT-V2 采用**分层多智能体系统（Hierarchical Multi-Agent System）**作为核心推理引擎。该架构将复杂的用户意图挖掘任务解耦为多个专业化智能体（如兴趣提取、场景理解、偏好推理等），通过顶层调度器进行协同路由与信息共享，避免 V1 版本中多推理路径并行导致的计算重叠。底层结合**混合表示推理（Hybrid Representation Inference）**模块，将离散的用户行为序列与连续的特征嵌入进行联合压缩，构建轻量级上下文表征，从而在维持深层语义推理能力的同时，大幅缩减显存占用与推理延迟。

### 关键技术
- **元提示框架（Meta-Prompting）**：摒弃静态模板，模型根据实时交互上下文、用户画像及商品属性动态组装提示词结构。该机制赋予模型更强的场景自适应能力，使生成的推荐理由更具个性化与逻辑连贯性。
- **约束强化学习（Constrained RL）**：针对标签预测准确性、解释合理性及商业目标之间的多奖励冲突，设计硬约束与软惩罚相结合的优化策略。通过策略梯度算法在可行域内搜索最优策略，确保模型在提升预测性能的同时不牺牲解释质量。
- **智能体裁判机制（Agent-as-a-Judge）**：将评估过程拆解为“事实核查-逻辑连贯性-用户价值对齐”多步推理链。裁判智能体模拟人类评审员的认知路径，对生成结果进行细粒度打分，有效缓解传统自动化指标（如 BLEU/ROUGE）与人类主观偏好脱节的问题。

## 实验结果
离线评估与线上 A/B 测试均验证了 RecGPT-V2 的显著优势：
- **计算效率与召回性能**：GPU 资源消耗降低 **60%**，独家召回率（Exclusive Recall）从 **9.39%** 提升至 **10.99%**，证明架构优化有效兼顾了效率与长尾覆盖。
- **生成质量与预测精度**：解释多样性指标提升 **+7.3%**；标签预测准确率提升 **+24.1%**，解释接受度提升 **+13.0%**，验证了元提示与约束 RL 的协同增益。
- **淘宝在线 A/B 测试**：核心业务指标全面超越基线模型，点击率（CTR）提升 **+2.98%**，商品详情页访问量（IPV）提升 **+3.71%**，总访问量（TV）提升 **+2.19%**，新客探索率（NER）大幅提升 **+11.46%**，展现出极强的商业转化潜力。

## 局限性
尽管 RecGPT-V2 在工程落地与效果上取得突破，但仍存在以下改进空间：首先，多智能体路由策略在极端稀疏或冷启动场景下的稳定性仍需增强，动态负载均衡机制有待完善；其次，约束强化学习的奖励函数设计仍依赖较多人工先验，自动化奖励建模与自对齐能力有限；最后，Agent-as-a-Judge 框架在复杂多轮交互或跨域推荐中的评估一致性仍需进一步校准，未来需探索更细粒度的人类偏好对齐与自监督路由优化技术。

## 与 LLM4Rec 的相关性
RecGPT-V2 是 LLM 赋能推荐系统（LLM4Rec）从“学术原型”迈向“工业级基础设施”的里程碑式工作。它直面 LLM 在推荐场景中普遍存在的推理冗余、生成僵化、评估失真及算力瓶颈四大痛点，提供了一套可复用的工程化技术栈。其多智能体协同推理、动态提示工程与约束强化学习的组合范式，为构建可解释、高效率、强泛化的意图驱动型推荐系统树立了新标杆。该工作不仅验证了 LLM 在大规模电商流量下的实时推理可行性，也为后续 LLM4Rec 研究在架构轻量化、多目标对齐及自动化评估方向提供了重要的实践参考与理论启发。
