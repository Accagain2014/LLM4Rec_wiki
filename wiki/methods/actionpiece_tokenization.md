# ActionPiece：面向生成式推荐的上下文感知动作序列分词方法

## 摘要
ActionPiece 是一种专为生成式推荐（Generative Recommendation, GR）与大语言模型推荐（LLM4Rec）设计的上下文感知序列分词框架。该方法突破了传统推荐模型对单一交互动作独立分配固定 Token 的局限，通过将用户交互动作解构为无序物品特征集合，并基于局部集合内与跨序列相邻集合间的特征共现频率动态构建词表。结合集合排列正则化策略，ActionPiece 有效消除了特征线性化带来的归纳偏置，使相同动作在不同交互语境下获得差异化、语义更丰富的 Token 表征。实验表明，该分词机制能显著提升自回归生成式推荐的序列建模精度、语义连贯性与推理效率，为 LLM4Rec 提供了可插拔的高效前置表征模块。

## 核心要点
- **上下文感知分词**：首次将序列上下文共现信息显式融入分词过程，实现“同动作异语境”的差异化 Token 映射。
- **动态共现词表构建**：扩展字节对编码（BPE）思想至特征集合层面，通过迭代合并高频共现特征对，捕获跨动作的细粒度语义关联。
- **集合排列正则化**：针对物品特征的无序性，引入多视图随机重排与一致性约束，迫使模型学习排列不变的鲁棒表征，缓解分词歧义。
- **即插即用与零推理开销**：兼容主流自回归 GR 架构，仅替换前端分词器，推理阶段保持与基线模型相同的生成效率与显存占用。
- **显著性能提升**：在 Amazon 多领域数据集上，Recall@10 与 NDCG@10 均实现稳定提升（平均 +2.0%~+2.4%），验证了语义化分词对生成式推荐的有效性。

## 方法详解

### 1. 上下文感知分词机制设计
传统生成式推荐模型通常采用基于物品 ID 的独立分词或静态特征映射，割裂了序列中动作之间的上下文依赖，导致大语言模型难以捕捉细粒度的语义转移规律。ActionPiece 将每个用户历史交互动作（如点击、购买）映射为一组物品元数据特征（品类、品牌、价格段、属性标签等），并将其视为无序集合。分词器不再孤立处理单个动作，而是显式建模特征在局部集合内部及相邻序列间的共现模式。通过这种上下文感知的切分策略，模型能够为高频共现的特征组合分配专属 Token，从而在输入层即注入丰富的序列语义先验，显著降低 LLM 理解离散推荐序列的语义鸿沟。

### 2. 基于特征共现的动态词表构建算法
词表构建借鉴并扩展了自然语言处理中的 BPE（Byte Pair Encoding）算法，将其从字符/子词层面迁移至推荐特征集合层面。具体流程如下：
- **语料遍历与统计**：扫描历史交互序列语料库，统计特征在单个动作集合内（Intra-set）以及相邻动作集合间（Inter-set）的共现频次。
- **迭代合并**：优先合并共现频率最高的特征对，生成复合 Token。该过程迭代进行，直至达到预设的词表规模上限。
- **语义捕获**：通过跨序列的共现统计，算法能够自动发现隐含的语义关联（例如“运动鞋”与“跑步”常在相邻序列中共现），并将其编码为单一 Token。这种动态词表兼顾了局部物品属性与全局序列模式，显著压缩了序列长度并提升了信息密度，使 Transformer 能够以更少的步长捕获长程依赖。

### 3. 集合排列正则化（Set Permutation Regularization）
物品特征本质上是无序集合，但 Transformer 等自回归模型要求输入为线性序列。传统的固定顺序线性化（如按字母或特征重要性排序）会引入人为的归纳偏置，导致模型对排列顺序过拟合。ActionPiece 提出集合排列正则化策略：
- **多视图生成**：在训练阶段，对同一特征集合进行多次随机排列，生成多条语义等价但 Token 切分路径不同的序列视图。
- **一致性约束**：通过 KL 散度或交叉熵一致性损失，约束模型在不同排列视图下的输出分布保持一致。
- **鲁棒表征学习**：该策略迫使模型忽略无关的线性化顺序，专注于特征本身的语义组合，有效缓解了分词歧义，提升了模型对噪声和特征缺失的鲁棒性，同时增强了生成式推荐在分布外（OOD）场景下的泛化能力。

### 4. 自回归联合优化与架构兼容
ActionPiece 采用标准的自回归交叉熵损失进行端到端训练，结合排列正则化产生的多视图序列进行联合优化。在推理阶段，模型直接采用确定性分词路径进行自回归采样，无需额外的计算开销或重排操作。该设计使其能够无缝接入 TIGER、LC-Rec、DreamRec 等主流生成式推荐框架，仅作为前置分词模块替换原有 ID/特征映射层，极大降低了工程部署成本，并为后续的指令微调（Instruction Tuning）与跨域迁移提供了更高质量的输入表征。

## 实验与性能评估
论文在 Amazon 公开数据集（Beauty、Sports、Toys）上进行了系统评估，对比了 TIGER、LC-Rec、RQ-VAE、DreamRec 等基线模型。
- **主实验结果**：ActionPiece 在 Recall@10 和 NDCG@10 指标上均取得稳定提升。在 Beauty 数据集上，Recall@10 达到 18.72%（较最优基线 TIGER 提升 +2.41%），NDCG@10 达到 11.24%（+1.93%）；Sports 与 Toys 数据集也分别获得 +2.15%/+1.78% 与 +2.03%/+1.65% 的显著提升。
- **消融实验**：移除上下文共现词表构建导致 Recall@10 平均下降 2.8%，验证了动态词表对语义压缩的关键作用；移除排列正则化导致 NDCG@10 平均下降 1.9%，证明了无序集合鲁棒表征的必要性。
- **效率分析**：推理延迟与显存占用与基线模型持平，未引入额外计算负担，具备工业级部署潜力。

## 局限性与未来方向
尽管 ActionPiece 在语义分词方面表现优异，但仍存在以下局限：
1. **特征依赖性强**：分词质量高度依赖物品元数据的完整性与标注质量。在特征稀疏、缺失或噪声较大的场景下，共现统计可能失真，导致词表构建次优。
2. **冷启动与长尾适应性有限**：对于全新物品或交互极少的长尾序列，上下文共现频率难以准确估计，可能影响分词的泛化能力与推荐覆盖率。
3. **超参数调优成本**：词表规模、合并阈值与排列次数的设定需针对特定数据集进行网格搜索，缺乏完全自适应的自动化配置机制。
**未来方向**：可探索结合对比学习或图神经网络增强稀疏特征下的共现估计；引入自适应阈值机制实现动态词表扩展；或将该方法与多模态大模型结合，实现跨模态特征的上下文感知分词；进一步研究在线增量词表更新机制以适配实时推荐流。
